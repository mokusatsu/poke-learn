# ドメイン知識と仕様 (Domain Logic)

本プロジェクト（`poke-learn`）固有の仕様や、ポケモンに関するドメイン知識の扱い方について説明します。

## 1. ポケモンデータの連携 (PokeAPI とローカルデータ)

本アプリケーションでは、ポケモンに関するマスタデータを取得・管理するために `PokeAPI` とローカルデータのハイブリッド構成を取っています。

- **`src/utils/pokemonApi.ts` (PokeAPI)**:
  - 基本的に PokeAPI (https://pokeapi.co/) を外部データソースとして利用し、ポケモンの基本情報（種族値、タイプなど）を取得します。
  - ネットワークリクエストのオーバーヘッドを減らすためのキャッシュ戦略が実装されています。
- **`src/data/localPokemons.ts` (ローカルデータ)**:
  - クイズに出題する特定のポケモンや、よく使われる対戦環境のポケモンの静的リストを保持しています。API への依存を減らし、高速な読み込みとオフラインに近い動作を担保するためのフォールバックおよびフィルタとして機能します。

## 2. タイプ相性の計算ロジック

ポケモンバトルにおける最も重要なドメイン知識が「タイプ相性」です。

- **`src/utils/typeMatrix.ts`**:
  - 全タイプ（ノーマル、ほのお、みず…等）の 18 x 18 の相性マトリクス（効果ばつぐん: 2.0倍、いまひとつ: 0.5倍、こうかがない: 0.0倍、等倍: 1.0倍）を定義しています。
  - 攻撃側のタイプと、防御側（複合タイプを含む）のタイプの組み合わせによる最終的なダメージ倍率を計算するロジック（`calculateEffectiveness` 等）が実装されています。
  - クイズコンポーネント（`MaxPowerQuiz` など）は、このロジックを呼び出して正解を判定します。

## 3. フレーバーテキスト（評価テキスト）の表記統一

クイズの正誤判定や、相性分析の UI においてユーザーに表示するフレーバーテキストは、実装者やエージェントによって表記揺れ（例：「ばつぐん」「抜群」「バツグン」など）が発生しやすいため、UI の一貫性を保つ目的で以下の表記に統一してください。

| 倍率 (Effectiveness) | 統一する表記 (日本語) |
| :--- | :--- |
| **2.0倍 以上** (2.0, 4.0) | `ばつぐん` （ひらがな） |
| **1.0倍** (等倍) | `等倍` または表示しない |
| **0.5倍 以下** (0.5, 0.25)| `いまひとつ` （ひらがな） |
| **0.0倍** (無効) | `こうかがない` または `無効` |

※ 原作ゲームのテキスト（例：「こうかは ばつぐんだ！」等）を文脈に合わせて使用することはUX向上に繋がりますが、単語として使用する際も「抜群」や「バツグン」といった表記揺れを避け、アプリケーション全体でトーン＆マナーを統一するための指示としてこのルールを厳守してください。
