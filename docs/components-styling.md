# UIコンポーネントとスタイリング (Components & Styling)

本プロジェクトにおける UI コンポーネントの実装およびスタイリングのルールについて説明します。

## 1. コンポーネント設計方針

- **ディレクトリ**: すべての UI コンポーネントは `src/components/` に配置します。
- **分割粒度**: コンポーネントが肥大化（概ね 300 行以上）した場合、再利用可能なサブコンポーネントとして分割することを検討してください。ただし、本プロジェクトのフェーズにおいて無暗な細分化は避け、実用性を重視します。
- **型定義**: Props には必ず TypeScript の Interface または Type Alias を定義し、明確に型付けを行ってください。

## 2. スタイリングの基本ルール

- **技術選定**: CSS Modules や TailwindCSS などの外部ライブラリは使用せず、**Vanilla CSS** と **Inline Styles** の組み合わせを基本とします。
- **グローバルスタイル (`index.css`)**:
  - アプリケーション全体で共通するテーマカラー（`--bg-color`, `--accent-violet` 等）やベーススタイルは、`src/index.css` に CSS 変数として定義されています。
  - カスタムクラスを定義する際は、他コンポーネントとの競合を避けるため、コンポーネント固有のプレフィックスを付けるか、DOM の階層関係を厳密に指定してください。
- **インラインスタイル**:
  - 動的に値が変わるスタイル（例：計算された位置、幅など）は React の `style` プロパティ（インラインスタイル）を利用します。

## 3. モダンなデザインの維持

エージェントが新規 UI を作成する際は、簡素なベタ塗りのデザインは避け、以下の**モダンな UI 要件**を満たすように実装してください。

- **グラスモーフィズム (Glassmorphism)**: 半透明の背景 (`rgba(...)`) と `backdrop-filter: blur(...)` を活用し、奥行きのある UI を実現すること。
- **マイクロアニメーション**: ボタンのホバー時や要素の表示・非表示時に、`transition` を用いた滑らかなアニメーション（`scale`, `opacity`, `translate` など）を実装すること。
- **適切な余白とシャドウ**: `box-shadow` で要素を浮き上がらせ、充分な Padding / Margin を取って窮屈にならないレイアウトにすること。

## 4. パフォーマンスとアニメーション

`[Do] (必須ルール)` としても定義されていますが、アニメーションや高頻度で切り替わる表示（ホバーエフェクト等）を実装する際の注意点です。

- **React State を使わない**: マウスの移動イベントなどに追従して React の State を更新し、インラインスタイルを書き換える実装は、再レンダリング処理が重くなるため避けてください。
- **CSS / GPU アクセラレーションの活用**: 動的なハイライト処理は `index.css` 内で疑似クラス（`:hover`）や、擬似要素（`::before`, `::after`）に対する `opacity` の操作として実装し、必ず `will-change: transform, opacity` などを付与して GPU によるコンポジットを促してください。
